# Requirements Document: Jan-Awaaz

## Introduction

Jan-Awaaz is a voice-first mobile application designed to help rural citizens in India access government schemes through voice interaction in major Indian regional languages. The system addresses critical barriers including language limitations, complex documentation requirements, lack of scheme awareness, and low digital literacy. The platform enables users to describe their situation verbally, verify documents through image capture, and receive personalized guidance on eligible schemes with clear directions to the correct government office.

## Glossary

- **System**: The Jan-Awaaz mobile application and backend services
- **User**: Rural citizen seeking government scheme assistance (primary user)
- **CSC_Operator**: Common Service Centre operator who assists with applications (secondary user)
- **Voice_Input**: Audio captured from user's microphone in major Indian regional languages (Hindi, Tamil, Telugu, Kannada, Malayalam, Marathi, Bengali, Gujarati, Punjabi, Odia)
- **Transcription_Service**: Amazon Transcribe service that converts voice to text
- **Voice_Output**: Audio response generated by Amazon Polly in local language
- **Document_Image**: Photo of physical document captured via mobile camera
- **Verification_Service**: Amazon Textract service that extracts and validates document data
- **Scheme_Matcher**: Amazon Bedrock Knowledge Base that matches user situations to schemes
- **Referral_Card**: Digital summary card (Parchee) showing scheme eligibility and requirements
- **Office_Locator**: GPS-based service that identifies nearest government office
- **Session**: User interaction tracked by mobile phone number
- **Green_Pin**: Map marker indicating CSC/Online application office
- **Red_Pin**: Map marker indicating Panchayat/Physical application office

## Requirements

### Requirement 1: Voice-Based User Input

**User Story:** As a rural citizen with low digital literacy, I want to describe my situation using voice in my regional language, so that I can access government schemes without typing or reading English.

#### Acceptance Criteria

1. WHEN a User speaks into the microphone, THE System SHALL capture the Voice_Input in any supported Indian regional language
2. WHEN Voice_Input is captured, THE Transcription_Service SHALL convert it to text within 5 seconds
3. WHEN the User speaks for more than 60 seconds continuously, THE System SHALL process the audio in segments without losing context
4. IF the Transcription_Service fails to process audio, THEN THE System SHALL prompt the User to speak again with Voice_Output guidance
5. WHEN Voice_Input contains background noise, THE System SHALL filter noise and extract the primary voice signal

### Requirement 2: Scheme Matching from User Story

**User Story:** As a rural citizen, I want the system to understand my life situation and match me to relevant government schemes, so that I don't need to know scheme names or eligibility criteria.

#### Acceptance Criteria

1. WHEN transcribed text is received, THE Scheme_Matcher SHALL query the Bedrock Knowledge Base with the user's situation
2. WHEN querying the Knowledge Base, THE Scheme_Matcher SHALL return only schemes that exist in the provided PDF documents
3. WHEN multiple schemes match the user's situation, THE System SHALL rank them by relevance and present the top 3 matches
4. WHEN no schemes match the user's situation, THE System SHALL respond with Voice_Output explaining no matches were found
5. WHEN a scheme match is found, THE System SHALL extract eligibility criteria, required documents, and application process details

### Requirement 3: Document Verification via Image Capture

**User Story:** As a rural citizen, I want to verify my documents before traveling to a government office, so that I don't waste time and money on trips that will be rejected.

#### Acceptance Criteria

1. WHEN a User captures a Document_Image via camera, THE System SHALL upload it to the Verification_Service
2. WHEN the Verification_Service processes a Document_Image, THE System SHALL extract text, dates, seals, and signatures within 5 seconds
3. WHEN an Aadhaar card is uploaded, THE Verification_Service SHALL validate the 12-digit number format and check for visible tampering
4. WHEN an Income Certificate is uploaded, THE Verification_Service SHALL check for required government seal and issuing authority signature
5. WHEN a document is expired based on extracted date, THE System SHALL notify the User via Voice_Output with the expiration date
6. WHEN a document is missing a required seal or signature, THE System SHALL notify the User via Voice_Output specifying what is missing
7. WHEN a Document_Image is too blurry or dark, THE System SHALL prompt the User to recapture with better lighting

### Requirement 4: Voice-Based Feedback and Guidance

**User Story:** As a rural citizen who cannot read, I want to receive all information through voice in my local language, so that I can understand the guidance without assistance.

#### Acceptance Criteria

1. WHEN the System generates a response, THE Voice_Output SHALL be produced in the same language as the Voice_Input
2. WHEN scheme matches are found, THE System SHALL use Voice_Output to describe each scheme in simple terms
3. WHEN document verification completes, THE System SHALL use Voice_Output to confirm validity or explain issues
4. WHEN Voice_Output is generated, THE System SHALL use Amazon Polly to synthesize speech within 5 seconds
5. WHEN the User requests information to be repeated, THE System SHALL replay the last Voice_Output

### Requirement 5: Location-Based Office Routing

**User Story:** As a rural citizen, I want to know exactly which government office to visit and how to get there, so that I go to the correct location on my first attempt.

#### Acceptance Criteria

1. WHEN a scheme requires physical application, THE Office_Locator SHALL identify the nearest Panchayat office and display a Red_Pin on the map
2. WHEN a scheme supports online application, THE Office_Locator SHALL identify the nearest CSC and display a Green_Pin on the map
3. IF Document Verification fails due to a missing physical attribute, THEN THE System SHALL automatically override the destination to Panchayat office with Red_Pin and explain via Voice_Output that a physical visit is required for correction
4. WHEN the User's GPS coordinates are available, THE System SHALL calculate distance to the nearest office within 10 kilometers
5. WHEN multiple offices are within range, THE System SHALL display all options ranked by distance
6. WHEN GPS coordinates are unavailable, THE System SHALL prompt the User to enable location services via Voice_Output

### Requirement 6: Digital Referral Card Generation

**User Story:** As a rural citizen, I want a simple visual card that I can show at the government office, so that the CSC operator understands what I need without me explaining everything again.

#### Acceptance Criteria

1. WHEN a User is matched to a scheme, THE System SHALL generate a Referral_Card containing scheme name, user phone number, required documents, and office location
2. WHEN the Referral_Card is displayed, THE System SHALL use heavy iconography alongside text to ensure comprehension by illiterate users
3. WHEN the Referral_Card is generated, THE System SHALL display it on screen with large, clear text and icons
4. WHEN the Referral_Card is displayed, THE System SHALL allow the User to save it as an image to their phone gallery
5. WHEN a CSC_Operator scans or views the Referral_Card, THE System SHALL display all eligibility criteria and document checklist
6. WHEN the Referral_Card is generated, THE System SHALL include a timestamp and unique reference number

### Requirement 7: Phone Number-Based Session Management

**User Story:** As a rural citizen without email or literacy skills, I want to use my phone number to access my information, so that I can return to my application without complex login processes.

#### Acceptance Criteria

1. WHEN a User first accesses the System, THE System SHALL request the mobile phone number via Voice_Output
2. WHEN a phone number is provided via Voice_Input, THE System SHALL create or retrieve a Session associated with that number
3. WHEN a User returns to the System, THE System SHALL recognize the phone number and restore the previous Session
4. WHEN a Session is restored and the status is PENDING_ACTION, THE System SHALL immediately prompt the User via Voice_Output with a contextual question without waiting for user input
5. WHEN a Session is created, THE System SHALL store scheme matches, document verification results, Referral_Cards, and action status
6. WHEN a Session is inactive for 30 days, THE System SHALL archive the data but retain the phone number association

### Requirement 8: Low Bandwidth Optimization

**User Story:** As a rural citizen with limited mobile data and slow internet, I want the app to work efficiently on my connection, so that I can complete my application without excessive data costs or timeouts.

#### Acceptance Criteria

1. WHEN uploading a Document_Image, THE System SHALL compress the image to under 500KB while maintaining readability
2. WHEN generating Voice_Output, THE System SHALL use compressed audio formats optimized for mobile networks
3. WHEN network connectivity is poor, THE System SHALL queue operations and retry automatically
4. WHEN the User is offline, THE System SHALL display a clear message via Voice_Output explaining the connectivity issue
5. WHEN data transfer is in progress, THE System SHALL display a visual progress indicator

### Requirement 9: Multi-Language Support

**User Story:** As a rural citizen who speaks a regional Indian language, I want to interact with the system in my native language, so that I can understand and be understood without language barriers.

#### Acceptance Criteria

1. WHEN the System starts, THE System SHALL detect the User's preferred language from device settings or prompt for selection
2. WHEN the User selects a supported language, THE Transcription_Service SHALL process Voice_Input in that language and THE Voice_Output SHALL be in that language
3. THE System SHALL support Hindi, Tamil, Telugu, Kannada, Malayalam, Marathi, Bengali, Gujarati, Punjabi, and Odia
4. WHEN the Referral_Card is generated, THE System SHALL display text in the User's selected language
5. WHEN the User switches language mid-session, THE System SHALL update all Voice_Output and text displays to the new language

### Requirement 10: Document Type Recognition

**User Story:** As a rural citizen who doesn't know technical document names, I want the system to automatically identify what document I'm showing, so that I don't need to select from confusing menus.

#### Acceptance Criteria

1. WHEN a Document_Image is uploaded, THE Verification_Service SHALL automatically classify it as Aadhaar, Income Certificate, Land Patta, or Other
2. WHEN the document type is identified, THE System SHALL apply type-specific validation rules
3. WHEN an Aadhaar card is identified, THE Verification_Service SHALL extract the 12-digit number, name, and date of birth
4. WHEN an Income Certificate is identified, THE Verification_Service SHALL extract annual income amount, issuing authority, and validity date
5. WHEN a Land Patta is identified, THE Verification_Service SHALL extract survey number, area, and owner name
6. IF the document type cannot be determined, THEN THE System SHALL prompt the User via Voice_Output to describe what document they are showing

### Requirement 11: Error Handling and Recovery

**User Story:** As a rural citizen with limited technical knowledge, I want clear guidance when something goes wrong, so that I can fix issues without getting stuck or frustrated.

#### Acceptance Criteria

1. IF the Transcription_Service fails to understand Voice_Input, THEN THE System SHALL ask the User to repeat more slowly via Voice_Output
2. IF the Verification_Service cannot process a Document_Image, THEN THE System SHALL explain the issue via Voice_Output and suggest recapture
3. IF the Scheme_Matcher returns no results, THEN THE System SHALL ask clarifying questions via Voice_Output to gather more information
4. IF network connectivity is lost during an operation, THEN THE System SHALL save progress and resume when connectivity returns
5. WHEN any service error occurs, THE System SHALL log the error with Session identifier for debugging without exposing technical details to the User

### Requirement 12: Scheme Information Accuracy

**User Story:** As a rural citizen relying on this system for critical benefits, I want accurate and up-to-date scheme information, so that I don't pursue schemes I'm not eligible for or miss ones I qualify for.

#### Acceptance Criteria

1. WHEN the Scheme_Matcher queries the Knowledge Base, THE System SHALL only return schemes that exist in the verified PDF documents
2. WHEN scheme eligibility criteria are presented, THE System SHALL display the exact criteria from the source document without modification
3. WHEN a scheme has been updated in the Knowledge Base, THE System SHALL reflect the changes in all new queries
4. THE System SHALL NOT generate or hallucinate scheme names, eligibility criteria, or application processes
5. WHEN a User asks about a scheme not in the Knowledge Base, THE System SHALL clearly state via Voice_Output that the scheme information is not available

### Requirement 13: Performance and Latency Requirements

**User Story:** As a rural citizen with limited patience for technology, I want quick responses to my voice commands, so that the interaction feels natural and I don't give up.

#### Acceptance Criteria

1. WHEN Voice_Input is captured, THE Transcription_Service SHALL return text within 5 seconds
2. WHEN the Scheme_Matcher processes a query, THE System SHALL return results within 5 seconds
3. WHEN a Document_Image is uploaded, THE Verification_Service SHALL complete analysis within 5 seconds
4. WHEN Voice_Output is generated, THE System SHALL begin playback within 2 seconds
5. WHEN the Office_Locator calculates routes, THE System SHALL display the map within 3 seconds

### Requirement 14: Privacy and Data Security

**User Story:** As a rural citizen sharing sensitive personal documents, I want my information to be secure, so that my identity and documents are not misused.

#### Acceptance Criteria

1. WHEN a Document_Image is uploaded, THE System SHALL encrypt it during transmission and storage
2. WHEN a Session is created with a phone number, THE System SHALL hash the number for storage
3. WHEN Voice_Input is processed, THE System SHALL not store raw audio beyond the active Session
4. WHEN a User's data is archived after 30 days, THE System SHALL remove all Document_Images and retain only scheme match history
5. THE System SHALL NOT share User data with third parties without explicit consent
